2  IRB Preparation

This section details the IRB preparation process for IIR studies.

2.1 Overview

1. You will receive an email from Larissa containing the study name, basic information, and a link to the project folder.

2. Locate the project folder in the S-drive. The folder should be pre-populated with all necessary files and templates.

3. Ultimately, the following documents must be completed (each form will be addressed individually below).

For a single-center study:

  • Protocol
  • Application Part 2
  • MOA
  • PI Attestation
  • PSD Form (only fill out if requested)
  • DOA
  • Data Sheet/Variable List (if not in protocol)

For a multi-center study in which we are the coordinating center:

  • Protocol
  • Application Part 2
  • MOA
  • PI Attestation
  • PSD Form (only fill out if requested)
  • DOA
  • Data Sheet/Variable List (if not in protocol)
  • Collaborative Research Supplement
  • Data Use Agreement (does not need to be submitted but must be completed prior to submission)

For a multi-center study in which we are a participating center:

  • Application Part 2
  • MOA
  • PI Attestation
  • PSD Form (only fill out if requested)
  • IRB Study Approval Letter (from coordinating center)
  • Protocol (from coordinating center)
  • In the unlikely event that our role in the study does not align with the coordinating center’s protocol, our own protocol is required.
  • Data Use Agreement (does not need to be submitted but must be completed prior to submission)

4. Locate the project application. It should be found in the folder titled “Project Proposal, Feasibility, Contract, Budget, Progress.”

5. Use information contained in the project application to fill out the protocol and application part 2. Reach out to study personnel listed in the project application for clarification if necessary (e.g. variable list, estimated sample size, etc.). Save the completed application part 2 to the “1 IRB Submissions” folder and the completed protocol to the “3 Protocol, DOA” folder.

6. Fill out necessary fields in PI Attestation, print a copy, and place it in a folder with the physician’s name on it and your name as the return address. Give the folder to Rhoda or Svetlana to deliver to the physician. Upon receipt of the signed form, scan and save the file to the “1 IRB Submissions” folder.

7. Fill out necessary fields in the MOA and send to Kris Fortman (KFortman@mhif.org). In the email, be sure to include the name of the study, the PI, and the following sections from the protocol:

  • Introduction - Background and Rationale
  • Study Objectives, Purpose, or Study Aims
  • Study Design

Upon receipt of the signed form, save the file to the “1 IRB Submissions” folder.

8. Fill out the DOA form using the list of study personnel from the project application.

  • The PI should be listed as “PI” with tasks 1, 2, 3, 4
  • Other MHI Physicians should be listed as “Co-I” with tasks 1, 2, 3, 4
  • The research coordinator should be listed as “RC” with task 1
  • Research scholars and research associates should be listed as “RA” with tasks 2, 3, 4
  • Ross, Larissa, and biostatisticians should be listed as “RA” with tasks 2, 3, 4, 5
  • The regulatory team should be listed as “REG” with tasks 18, 19

9. Upon completion of all necessary forms, email Laura (laura.winters@allina.com) to let her know IRB prep is complete.

2.2 Protocol

1. A protocol template should be pre-populated in the “3 Protocol, DOA” folder. If it is not, the template can be found at in the S drive under “Metro -> ANW -> Research -> Coordinator & Regulatory Resources -> Templates - All Types -> IIR Templates, Checklists, and other Tools -> Protocol Templates”.

2. Rename protocol with study short title (e.g. Cardiopregnancy Protocol)

3. Open protocol template and follow the italicized directions on the protocol template. Most necessary information should be contained in “IIR Project Application” (study proposal) located in the “2 Project Proposal, Feasibility, Contract, Budget, Progress” folder. The project application should contain all necessary information for the following sections:

  • Introduction
  • Study Objectives
  • Study Design
  • Selection of Subjects
  • Data Elements
  • Methods

If any information is missing, contact research coordinator or other study personnel for guidance. Study personnel, including the research coordinator, are listed in the study proposal.

4. Delete instructions upon completion (this is includes bolded text and bullet points found throughout the protocol).

5. Important aspects of each protocol to pay attention to:

  • Is the study retrospective, prospective, or both? Please clearly state this information in the “Study Design” section.
  • What are the dates of investigation? Please clearly state the date range in which patients will be identified in the “Study Design” section, including any follow-up times.
  • What is the number of patients expected? Please use descriptors like “around” or overestimate to allow for flexibility.
  • Will the data be stored in Excel or REDCap? Please choose stock language accordingly.
  • What is the justification for a HIPAA waiver? Please choose based on sample size, either less than 50 or greater than 50.
  • What is the benefit of the study for study participants? Please use stock language in the protocol and adjust for the specific study.
  • Does the study utilize pre-existing databases or registries? If using data from another study, or a preceding registry, include the IRB number of that study and include this in the section “Research Databases, Existing Study Data Sets or Registries to be Utilized”
  • The variable list should be included in “Appendix A: Data Collection Form.” If no variable list is present, please reach out to the research coordinator for a copy.

6. Other notes:

  • Delete instructions at the end of the protocol after the appendix form.
  • De-italicize any stock language that has been kept.
  • Update the footer to include short title, protocol version, and date.
  • Confirm protocol is saved in the folder titled “3 Protocol, DOA”.
  • If you are unsure how to proceed, consult a similar study’s protocol as a guide. Alternatively, reach out to another RA or a member of the regulatory team.
  • Upon completion, send the protocol to the PI and have them review for accuracy.

2.3 Application Part 2

1. A template for Application Part 2 should be pre-populated in the “1 IRB Submissions” folder. If it is not, the template can be found at IRBnet.org -> Forms and Templates. For a chart review study, use “Form - Application 2 - Retrospective/Prospective Chart Reviews.”

2. Some sections of Application Part 2 contain the same information as the protocol. Please refer to the chart below to fill out relevant sections:

Application Part 2 Section Corresponding Protocol Section
  1. Data Sources and Confidentiality, a.
  1. Methods, A. Data Source
  1. Data Sources and Confidentiality, d.
  1. Methods, C. Data Transfer & Storage AND D. Data Sources and Confidentiality
  1. Data Sources and Confidentiality, f., ii.
  1. Methods, A. Data Source
  1. HIPAA Authorization, b., i.
  1. HIPAA Authorization (HIPAA Authorization - Waiver Request and Justification)
  1. HIPAA Authorization, b., ii.
  1. HIPAA Authorization (HIPAA Authorization - Waiver Request and Justification)
  1. HIPAA Authorization, b., iii.
  1. Methods, E. Data Retention and Destruction

3. Save completed form to the project folder in “1 IRB Submissions.”

2.4 PSD (Project Specific Disclosure)

NB The PSD form is no longer required for submission unless specifically requested by the IRB.

1. Download from IRBnet.org: Forms and Templates -> Form – COI – Project Specific Disclosure Form.

2. Fill in Study Title and PI.

3. Email to PI to fill out. Most answers will be NA or No, except to the last question, which should be a yes. However, there could be a test article, etc.

4. Save to the project folder in “1 IRB Submissions.”

2.5 IIR PI Attestation

1. A template for the IIR PI Attestation should be pre-populated in the project folder under “1 IRB Submissions.” If there is no template in the project folder, a blank PI Attestation can be found in the S-drive under Metro -> ANW -> Research -> Coordinator & Regulatory Resources -> Templates - All Types -> IIR Templates, Checklists and other Tools -> New Study Applications -> IIR PI Attestation.docx.

2. Fill in study title and PI.

3. Print form and place in a folder. On the outside of the folder, include the PI’s name and your own. For example, “To: Dr. Eckman” and “Please return to Ronald Thao, 6th floor.”

4. Give the form to Svetlana. She will deliver it to the PI for his/her signature and will return it when it has been signed.

5. Scan the form to a PDF and save to the project folder in “1 IRB Submissions.”

2.6 Memo of Acknowledgement (MOA)

1. A template for the MOA should be pre-populated in the project folder under “1 IRB Submissions.” If there is no template in the project folder, a blank MOA can be found in the S-drive under Metro -> ANW -> Research -> Coordinator & Regulatory Resources -> Templates - All Types -> IIR Templates, Checklists and other Tools -> New Study Applications -> IIR MOA_Blank.docx.

2. Fill in study title and PI.

3. Email the signable PDF MOA to Kris Fortman. Title the email “MOA Signature Required” and include a study synopsis. The synopsis can be taken from the following sections of the protocol:

  • Introduction - Background and Rationale
  • Study Objectives, Purpose, or Study Aims
  • Study Design

4. Upon receipt of the signed MOA, save to the project folder in “1 IRB Submissions.”

2.7 DOA (Delegation of Authority)

1. A template for the DOA should be pre-populated in the project folder under “3 Protocol, DOA.” If there is no template in the project folder, a blank DOA can be found in the S-drive under Metro -> ANW -> Research -> Coordinator & Regulatory Resources -> Templates - All Types -> IIR Templates, Checklists and other Tools -> DOA Template and Work Standard -> DOA General IIR retro blank.doc.

2. Add study name and sponsor.

3. Add PI and other co-investigators listed on the intake form.

4. Add the study coordinator (refer to previous projects for the PI to see who their coordinator typically is).

5. Add IIR team as RAs.

6. Add regulatory team as REG.

7. Code each participant’s role as described below:

  • The PI should be listed as “PI” with tasks 1, 2, 3, 4
  • Other MHI Physicians should be listed as “Co-I” with tasks 1, 2, 3, 4
  • The research coordinator should be listed as “RC” with task 1
  • Research scholars and research associates should be listed as “RA” with tasks 2, 3, 4
  • Ross, Larissa, and biostatisticians should be listed as “RA” with tasks 2, 3, 4, 5
  • The regulatory team should be listed as “REG” with tasks 18, 19

8. Save completed form to project folder in “3 Protocol, DOA.”
